Connexion à l'hôte, login etc.

"Login"

Pour accéder au MOO il faut d'abord choisir un moyen d'accès (voir ci-dessous). Ensuite il faut se "logger" en rentrant la commande suivante dans un client MOO (substituez "mot-de-passe" par votre mot de passe).
  • Acces "normal": 
        connect TSR mot-de-passe
  • Acces comme un enfant: 
        connect TSRKid mot-de-passe
Notre MOO se trouve sur l'hôte: tecfamoo.unige.ch , porte: 7777. Si jamais votre DNS est en panne voici le numéro IP: 129.194.12.39.

Accès avec un applet JAVA

Nécessite aucune installation de votre part (sauf avoir Netscape 2.x ou mieux). Il s'agit de "clients" simples, mais utilisables. Dès que vous clickez sur un applet ci-dessous il est téléchargé dans votre browser et la connextion s'établit avec notre MOO. Il suffit de rentrer les commandes dans la petite ligne blanche pour input. Il faut commencer une séance en tapant "connect TSR ...." (voir ci-dessus).
  • Surf'n Turf (recommandé)
  • MudSlide (si Surf'n Turf ne marche pas)
  • JMUd (si les autres ne marchent pas)

Accès via le WWW

L'accès via le Web permet de regarder les objets du MOO et parfois de les manipuler. Cet interface n'est pas utile pour communiquer !!

Clients suggérés

Pour Windows
  • MUDWIN: Il s'agit d'un client simple (comme utilisé par les élèves de Malagnou.). Il semble avoir un problème de fontes avec Win95.
    • Il faut telecharger les fichiers qui sont ici: ftp://tecfa.unige.ch/pub/software/win31/mud-clients/mudwin/
      (Ou alternativement telecharger l'archive mudwin.zip si vous possédez le logiciel "WinZip".
    • Pour ouvrir une connection avec un MOO il faut utiliser le menu "file", l'option NEW (ou OPEN les prochaines fois): 
         HOST:   tecfamoo.unige.ch
   PORT:   7777
    • Ensuite on rentre les commandes dans la petite fenetre blanche.
  • Si vous avez un informaticien à disposition, je suggère l'installation du client tkMOO light (environ 1 heure de travail).

Commandes de base

Notez que la plus part des gens apprennent à MOOer par immersion. On leur donne qq commandes de base et ensuite ils apprennent en "faisant" dans le MOO.
Commandes de base: 

say                                  - parler dans une salle -
     say bonjour, comment ca va ?
     "bonjour, comment ca va ?
page                                 - parler à distance ou en secret -
     page Kaspar on va manger ?
emote                                - "poser", montrer des emotions -
     emote rigole
     :rigole
@join                                - joindre une personne -
     @join Kaspar
go                                   - aller qq part -
     go up 
     up
look                                 - regarder des objets -
     look Daniel

Quelques remarques techniques

Un "MOO" est un serveur de réalité virtuelle textuelle (RVT). Il s'agit d'une sorte de combinaison entre les forums du Télétexte (ou Compuserve) et les jeux d'aventures textuelles. Imaginez d'être à l'intérieur d'un livre vivant. Cette technologie est à nos yeux (et sur le plan fonctionnel) le précurseur le plus important du futur "cyberspace". Un MOO peut être caractérisé de la façon suivante (sorry for some English):
  • Beaucoup de gens peuvent se connecter à un server (ou plus proprement parlé à un caractèrs ou personnage ou avatar)
  • Le MOO possède une organisation spaciale, vous interagissez avec d'autres personnes ou les objets principalement dans des "salles" (rooms) qui sont en principe connectés entre elles par des "portes".
  • Un "monde" MOO est programmé depuis l'intérieur. Cela veut dire que depuis le monde entier les gens peuvent se connecter et transformer ce monde en y rajoutant des objets et/ou des nouvelles fonctionalités.
  • Within a MOO, many real time communication actions exist, like "saying" or "emoting" things publically to the persons in a "room", paging people elsewhere, "whispering" messages, using a "CB" channel, etc.
  • Asynchronous communication tools include internal e-mail, Newsgroups, News papers, tutorial rooms, "white boards", etc.
  • Artifacts include talking "Robots", "specially designed rooms for teaching or discussion", "transport vehicles, notes and books, and anything that can be imagined and programmed.
